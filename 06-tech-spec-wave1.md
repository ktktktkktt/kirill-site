# ТЗ на реализацию первой SEO-волны

## 1) Цель
Реализовать первую волну SEO-посадочных в текущем Next.js проекте для продвижения в Яндекс и Google по кластерам разработки сайтов.

## 2) Границы релиза
- Типы страниц:
  - главная,
  - услуга,
  - услуга + ниша,
  - услуга + ниша + город.
- Первая волна:
  - 3 услуги,
  - 3 ниши,
  - 5 городов.

## 3) Состав данных (источник контента)
- Справочники: услуги, ниши, города.
- SEO-данные: шаблоны title/description/keywords.
- Контент-данные: боли, выгоды, блоки, FAQ, офферы.
- Коммерческие параметры: цена от, срок, CTA.

## 4) Требования к роутингу
- Поддержать URL-структуру:
  - `/service/{serviceSlug}`
  - `/service/{serviceSlug}/{nicheSlug}`
  - `/service/{serviceSlug}/{nicheSlug}/{citySlug}`
- Для массовых страниц использовать статическую генерацию (`generateStaticParams`).
- Невалидные параметры должны отдавать `404`.

## 5) Требования к SEO
- На каждой странице:
  - уникальный `title`,
  - уникальный `description`,
  - один `H1`,
  - self-canonical.
- Обновить `sitemap.xml` и `robots.txt` под боевой домен.
- Добавить Open Graph и schema.org по правилам из `04-seo-rules.md`.

## 6) Требования к контенту
- Следовать брифам из `03-template-briefs.md`.
- Для страниц `услуга+ниша+город` обеспечить уникализацию не менее 35%.
- Добавить внутреннюю перелинковку:
  - вверх по иерархии,
  - в соседние города,
  - в смежные услуги.

## 7) Требования к аналитике
- Подключить события из `05-kpi-analytics.md`:
  - `lead_submit`,
  - `lead_click_phone`,
  - `lead_click_messenger`,
  - `lead_click_cta_primary`,
  - `scroll_75`.
- Подготовить базовую сегментацию по типу страниц.

## 8) Критерии приемки
- Успешный `npm run build`.
- Все страницы первой волны генерируются и доступны.
- В sitemap присутствуют URL первой волны.
- Нет дублей title/description в первой волне.
- Формы и CTA отправляют события аналитики.
- Базовая перелинковка работает и не создает битых ссылок.

## 9) Порядок реализации
1. Обновить структуру данных услуг/ниш/городов.
2. Реализовать роуты и генерацию страниц по новой URL-схеме.
3. Внедрить метаданные и служебные SEO-файлы.
4. Добавить контентные блоки и перелинковку.
5. Подключить аналитику и события.
6. Проверить build, sitemap, индексацию и внутренние ссылки.
